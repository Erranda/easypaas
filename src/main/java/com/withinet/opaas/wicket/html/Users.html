Frequently Asked Questions

  1. How to start your platform as a service scenario with EasyPaaS?

    Download EasyPaaS and run with Java 1.6 minimum as java -jar easypaas-1.0.0.jar

  2. How do I configure the system?

    Once you run the main jar file, the auto configuration component will request some basic information relating the application configuration e.g your administration account details

  3. How secure is the infrastructure?

    All instances are sandboxed with policies that restrict cross-instance attacks for read,write operation on the file system. Only relative directories are permitted within the system. Future release will allow extending the default policy.

  4. What are the system limits?

    All instances require a port number for communication. During configuration you will be asked to specify a port range for the host server, this range inherently limits the number of instances deployable on the system. If your instance is taking long to deploy there's a chance you have run out of available ports.

  5. Why are my team members not receiving their account details?

    The system has an email component which can be configured during start-up or later in the application.properties file of the launch working directory. Ensure the information provided is correct.

  6. Some components are not visible in my view?

    An administrator can control what you see in your view based on roles and permission. Ensure your Administrator has given you the appropriate permissions on the system.

  7. Something went wrong and I get the Ooops page?

    This is an internal server error, check your application.properties file and ensure all your configuration details are systematically and syntactically correct

  8. Application persistence?

    The application by default uses an embedded database server which last the lifetime of each run. If you require full persistence start a database server compatible with Java and add its configuration at start up or in the application.properties file.
