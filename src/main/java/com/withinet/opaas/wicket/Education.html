Teaching Collaborative Software Development is possible with existing technologies. This system offers a great computing abstraction for managing large scale projects in academia.

About OSGi Cloud

Servlet Containers

Students are given continuous running application containers with a set of project components in which they are expected complete its lifecycle with through a web interface. A lot of the configuration complexities involved in setting these is not exposed to either the lecturer or the student. This allows the curriculum be structured in a way that suits the current skills of the students.

Control Panel

Configuration changes are managed at deployment time. For example, you could have 12 students running an instance of project in their own platform containers. If a core component of this project is updated, the student instances also get updated without the need for code distribution through a source control system or a plain old file system operation.

Provisioning System

So you have some of your project bundles already defined in form a maven build or a set of jar files. Just provide these to us and we will handle the business of making them available in your platform containers.

Collaboration

With fine grained roles authorization. You can provide your team members with different view points of the system.

Multi-tenant Security

Your instances run in a sanboxed environment to prevent malicious activity

Open-source

Built with open source technologies, we provide you different features to help kick start your software engineering curriculum.
